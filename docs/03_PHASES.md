# 개발 페이즈 및 순서

## Phase 1 — 기반 + 핵심 (프로젝트 보드 + 대화형 입력)

> 모든 기능의 중심. 이게 없으면 나머지가 의미 없음.

### Phase 1-1: 프로젝트 셋업
- [ ] Next.js App Router + TypeScript + Tailwind + shadcn/ui 초기 세팅
- [ ] Supabase 프로젝트 생성 (DB + Auth + Storage)
- [ ] 인증: Supabase Auth (이메일 또는 소셜 로그인) — 성훈, 태수 2명
- [ ] 기본 레이아웃: 사이드바 네비게이션 + 메인 콘텐츠 영역
- [ ] 반응형 (모바일에서도 사용 가능하게 — 현장에서 폰으로 볼 수 있어야 함)

### Phase 1-2: 프로젝트 보드 (F01)
- [ ] DB 스키마: spaces(공간), items(항목), decisions(결정사항)
- [ ] 공간 CRUD (추가/편집/삭제)
- [ ] 항목 CRUD (각 공간 하위)
- [ ] 매트릭스 뷰 UI: 공간(행) × 항목(열) 테이블
- [ ] 셀 클릭 → 상세 편집 모달 (상태, 내용, 메모, 사진 첨부)
- [ ] 상태 관리: 미결정/후보있음/결정됨/구매완료/시공완료
- [ ] 공간별 상세 뷰: 카드 형태
- [ ] 전체 진행률 대시보드

### Phase 1-3: Q&A + 대화형 입력 (F04)
- [ ] 채팅 UI (메시지 리스트 + 입력창)
- [ ] Claude API 연동
- [ ] Intent 분류 시스템: UPDATE vs QUESTION vs NOTION_ANALYZE
- [ ] UPDATE intent → 프로젝트 보드 자동 반영 로직
  - 공간/항목 자동 매칭
  - 사용자 확인 후 반영 ("화장실 세면대를 '분청 도자기'로 결정함으로 기록할까요?")
- [ ] QUESTION intent → 웹 검색 + LLM 답변
- [ ] 채팅 이력 저장 (Supabase)

## Phase 2 — 장소 탐색 + 시공업자

> 1순위 니즈. 보드에 결정사항이 쌓이면 바로 필요.

### Phase 2-1: 장소 탐색 + 스케줄 (F02)
- [ ] 장소 검색 UI: 검색바 + 결과 리스트 + 지도
- [ ] 지도 API 연동 (Naver Maps 또는 Kakao Maps)
- [ ] 검색: 웹 검색 API 또는 Places API 활용
- [ ] 장소 카드: 이름, 주소, 전화번호, 영업시간, 거리
- [ ] 장소 저장 (즐겨찾기)
- [ ] 스케줄 짜기: 여러 장소 선택 → 동선 최적화 → 일정 생성
- [ ] 일정 뷰: 시간대별 방문 계획 카드
- [ ] 방문 후 기록: 완료 체크, 사진, 메모
- [ ] LLM 기반 자동 추천: 프로젝트 보드 항목 기반 쇼룸 검색 키워드 생성

### Phase 2-2: 시공업자 관리 (F09)
- [ ] 업자 프로필 CRUD
- [ ] 견적서 파일 첨부 + 금액 기록
- [ ] 견적 비교 테이블 뷰
- [ ] 연락/방문 이력 기록
- [ ] 프로젝트 관리 F03 연결 (업자 → 태스크 배정)

- [ ] 드래그앤드롭 정렬
- [ ] 메모 기능
- [ ] 공유 링크 생성
